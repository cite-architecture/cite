Package: edu.harvard.chs.cite

[Groovy] Class TextInventory

    • edu.harvard.chs.cite.TextInventory

    • Class modelling a TextInventory. Methods for finding artifacts in the TextInventory appear in two forms:

      • an internal "private" method returning a TextInventory class data structure. These methods have 'Data' in their names.
      • a public method returning URN values or String values
      The internal 'Data' methods should not be used: internal data structures may change. The URN and String methods are stable. They will not change their signature and should produce identical results even if the underlying implementation changes.

        • void checkDataValues(groovy.util.Node root)

          Verifies that data values in the inventory comply with a number of constraints of the CTS specification. These include:

          • All URNs must be properly placed in their inherited hierarchy
          • A CTS namespace must be declared for all text groups
          • xml:lang attributes should be 3-char abbrs and xml:lang for translations should not be equal to the xml:lang value for the notional work
          • recursively checks the XPath values of scope and xpath attributes on all citationMappings and verifies that all XML namespaces are declared.
